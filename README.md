# C* (C-star) Language — Alpha Release

**C*** — это системный язык программирования, рождённый из отказа от неопределённости. В то время как другие языки стремятся к простоте через «магию», C* стремится к надёжности через **явность** и **строгий контроль**.

> "Почему мы соглашаемся с тем, чего не понимаем?" — Философия C*.

Вы пишете в три раза больше кода. Компилятор даёт вам в три раза больше гарантий.

---

##  Ключевые особенности (Alpha)

Компилятор прошёл стадию прототипа и теперь реализует уникальную систему **Level Traversal** (Stage 4), которая гарантирует безопасность ещё до выполнения кода.

**Система уровней (Level System)**
Компилятор строит граф путей (CFG) и проверяет «узкие места» на всех путях выполнения.

**Гарантированное владение (Ownership)**
Механика `as owner`, `as access` и `as copy` предотвращает использование переменных после передачи владения — как в Rust, но с явным синтаксисом.

**Безопасность памяти (Memory Safety)**
Обязательная проверка на `null` перед разыменованием. Автоматический поиск утечек: компилятор заблокирует сборку, если `release` для данных на куче не вызывается на всех путях выполнения.

**Инварианты типов (Range Check)**
Проверка диапазонов значений (например, `Health` от 0 до 100) на этапе компиляции — не в рантайме.

**Объектно-ориентированный подход**
Поддержка структур (`define type`) и полноценных методов с неявной передачей `%self`.

**Битовая точность**
Специальный синтаксис `bits of ... shifted` для написания низкоуровневых драйверов.

##  Пример кода

```cstar
program GrandMaster

define type[Health] as integer[signed, 32bit]
    valid range from 0 to 100
end type

function calculate_factorial
    receives integer[signed, 32bit] named n
    returns integer[signed, 32bit] named result
    body
        if n is less or equal 1
            result is 1
        otherwise
            result is n multiply by calculate_factorial receives n minus 1
        end otherwise
        end if
    end body
end function

body
    declare Health named hero_hp is 100
    -- hero_hp is 150  <-- Ошибка компиляции: вне допустимого диапазона

    declare array[integer[signed, 32bit], 3] named scores on heap is 0
    scores at index 0 is 10
    release scores  -- Обязательно: компилятор проверит наличие на всех путях
end body
```

##  Архитектура компилятора

Компиляция в C* прозрачна и состоит из 7 стадий (Раздел 6 спецификации):

| # | Стадия | Описание |
|---|--------|----------|
| 1 | Lexical Analysis | Разбор символов в токены (UTF-8) |
| 2 | Syntax Analysis | Строгий парсер (правило «один блок — одно закрытие») |
| 3 | Type Checking | Полный семантический анализ без неявных преобразований |
| 4 | **Level Traversal** | **Статическая верификация: владение, время жизни, null-safety** |
| 5 | Optimization | Удаление мёртвого кода, свёртка констант |
| 6 | Code Generation | Генерация LLVM IR |
| 7 | Linking | Транспиляция в C и финальная сборка через `gcc` |

##  Как запустить

Требования: **Python 3** и **gcc/clang**  
В Termux: `pkg install clang python`

```bash
# 1. Клонировать репозиторий
git clone https://github.com/shellwex/cstar-lang.git
cd cstar-lang

# 2. Запустить пример (компиляция + выполнение)
python3 compiler/cstarc.py examples/grand_master.cstar --run
```
